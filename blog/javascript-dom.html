An introduction to DOM

The purpose of this article is to introduce the Document Object Model (or DOM) and how it relates to JavaScript and its interactions with HTML and CSS.

Before we delve further into the DOM, lets discuss how JavaScript, HTML and CSS work together. I like to think of this in the context of a film. You have your script (HTML) which contains the content of the film, you have a costume/stage department (CSS) who are in charge of the look/feel of the actors and the set, and finally, you have the cool kids: the special effects department (JavaScript) who have the ability to add graphics, animations, styles, and even completely replace elements (pun intended) of the film with brand new frames.

Control Flow and Loops

The Control Flow is simply the order in which the system executes statements in a script. Code will run in order from the start to the finish unless it is asked to change the control flow by a command structure such as a conditional or a loop. There are 5 main types of command structures that allows this sort of manipulation

Conditional Statements

  • if statements
  • if.. else.. statements

Looping Statements

  • while loops
  • do.. while loops

switch statements

label statements

with statements

If I were to liken this to an everyday activity, I would compare this to washing your car. Simply deciding whether or not to wash the car is a simple example of an: IF statement. For example, IF the car is dirty, we should wash it, otherwise we dont.

In terms of loops, if we use the same analogy of washing a car, we could think of this as the control process to decide when to stop scrubbing the wheels for example. The 3 main components we need to complete the loop setup include a start point, an end point, and what to do at the end of each loop, you can then define what actions are taken whilst inside that loop (with the exception of a while loop, which only has 1 conditional requirement).

The Document Object Model (DOM)

The DOM can be thought of as a way of categorising the structure of elements of a webpage into objects that are then accessible by JavaScript. These objects can then be modified via code in Javascript which will allow for dynamic changes to a website without the need to refresh the page. This can be visualised by thinking of the dependencies between different elements like a family tree, where there are parent and child elements, descending from a start point (which in the case of DOM is usually window).

The Document portion of this tree sits one level below Window and gives us the gateway we need to access any child elements.

To understand how this can be valuable, think about how powerful the ability is to not only access a portion of a website, but modify it according to user inputs, or login status, or even based on the position of a users mouse (plus many more variables!). The developer can even incorporate complex functions that modify multiple elements on a page, completely changing what is seen/experienced by the user.

An example of how you could interact with the DOM is as follows: Say you have a website that caters for both dog lovers and cat lovers, we could alter how the page appears (whether that's with a puppy background, or in an ecommerce focus, we could populate a set area of the page to display dog/cat related items) to the logged in user without changing the structure of the HTML code.

Arrays vs Objects

Within Javascript, it is common practice to store information in a list format, this is usually achieved using an array, which is accessed via it's numerical index position (starting from 0) Arrays can store multiple types of values, such as strings, numbers, boolean etc.

Objects are quite similar to Arrays in that they store information in a list or table format, however, objects are known as key value pairs, where the value is accessed using the "key" of that given value. Objects can be indexed like an array, where the key is used as the index.

Functions

Functions give us the ability to program a set of instructions that can then be called upon as many times as required within the code. This is particularly useful where certain actions are required to be used multiple times within one document.

There are 2 parts to a function, defining it, and calling it. Defining the function tells the system what instructions are to be carried out, and calling the function tells the system to execute that set of instructions.
